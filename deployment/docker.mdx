Deploy MiroFish quickly using Docker and Docker Compose. This method provides a containerized environment with all dependencies pre-configured.

## Prerequisites

Before starting, ensure you have the following installed:

- **Docker**: Version 20.10 or higher ([Installation Guide](https://docs.docker.com/get-docker/))
- **Docker Compose**: Version 2.0 or higher (included with Docker Desktop)

Verify your installation:

```bash
docker --version
docker compose version
```

## Quick Start

<Steps>
  <Step title="Clone the Repository">
    Clone the MiroFish repository to your local machine:

    ```bash
    git clone https://github.com/666ghj/MiroFish.git
    cd MiroFish
    ```
  </Step>

  <Step title="Configure Environment Variables">
    Copy the example environment file and configure your API keys:

    ```bash
    cp .env.example .env
    ```

    Edit the `.env` file with your preferred text editor:

    ```bash
    nano .env
    # or
    vim .env
    ```

    **Required Configuration:**

    ```env
    # LLM API Configuration (supports any OpenAI SDK compatible API)
    # Recommended: Alibaba Bailian qwen-plus model
    # Sign up at: https://bailian.console.aliyun.com/
    LLM_API_KEY=your_api_key_here
    LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
    LLM_MODEL_NAME=qwen-plus

    # Zep Cloud Configuration for memory and graph storage
    # Free tier available at: https://app.getzep.com/
    ZEP_API_KEY=your_zep_api_key_here
    ```

    **Optional Boost Configuration:**

    ```env
    # Acceleration LLM Configuration (optional)
    LLM_BOOST_API_KEY=your_api_key_here
    LLM_BOOST_BASE_URL=your_base_url_here
    LLM_BOOST_MODEL_NAME=your_model_name_here
    ```

    <Warning>
      Never commit your `.env` file to version control. It contains sensitive API keys.
    </Warning>
  </Step>

  <Step title="Pull and Start the Container">
    Use Docker Compose to pull the latest image and start the service:

    ```bash
    docker compose up -d
    ```

    This command will:
    - Pull the latest MiroFish image from GitHub Container Registry
    - Create and start the container in detached mode
    - Automatically load environment variables from `.env`
    - Map ports 3000 (frontend) and 5001 (backend) to your host

    <Tip>
      If you experience slow download speeds in China, edit `docker-compose.yml` and uncomment the mirror image:
      
      ```yaml
      # image: ghcr.io/666ghj/mirofish:latest
      image: ghcr.nju.edu.cn/666ghj/mirofish:latest  # Mirror for faster downloads
      ```
    </Tip>
  </Step>

  <Step title="Verify Installation">
    Check that the container is running:

    ```bash
    docker compose ps
    ```

    You should see the `mirofish` container with status "Up".

    Access the application:
    - **Frontend**: http://localhost:3000
    - **Backend API**: http://localhost:5001
  </Step>
</Steps>

## Docker Configuration Details

### Port Mappings

The Docker Compose configuration exposes the following ports:

| Service | Container Port | Host Port | Description |
|---------|---------------|-----------|-------------|
| Frontend | 3000 | 3000 | Web interface |
| Backend API | 5001 | 5001 | REST API endpoints |

To change the host ports, edit `docker-compose.yml`:

```yaml
ports:
  - "8080:3000"  # Access frontend on port 8080
  - "8081:5001"  # Access backend on port 8081
```

### Volume Mounts

The configuration includes a persistent volume for uploaded files:

```yaml
volumes:
  - ./backend/uploads:/app/backend/uploads
```

This ensures that:
- Uploaded seed materials persist across container restarts
- Files are accessible on your host system in `backend/uploads/`
- Simulation data is preserved

### Restart Policy

The container is configured with `restart: unless-stopped`, meaning:
- Automatically restarts if it crashes
- Restarts on system reboot
- Only stops when explicitly stopped with `docker compose down`

## Managing the Container

### View Logs

Monitor application logs in real-time:

```bash
docker compose logs -f
```

View logs for the last 100 lines:

```bash
docker compose logs --tail=100
```

### Stop the Container

```bash
docker compose stop
```

### Start a Stopped Container

```bash
docker compose start
```

### Restart the Container

```bash
docker compose restart
```

### Stop and Remove the Container

```bash
docker compose down
```

<Warning>
  Using `docker compose down` removes the container but preserves volumes. Your uploaded files in `backend/uploads/` will remain intact.
</Warning>

## Updating MiroFish

To update to the latest version:

<Steps>
  <Step title="Pull the Latest Image">
    ```bash
    docker compose pull
    ```
  </Step>

  <Step title="Recreate the Container">
    ```bash
    docker compose up -d
    ```

    Docker Compose will automatically recreate the container with the new image.
  </Step>

  <Step title="Clean Up Old Images (Optional)">
    Remove unused Docker images to free up space:

    ```bash
    docker image prune -a
    ```
  </Step>
</Steps>

## Building from Source

If you want to build the Docker image locally instead of pulling from the registry:

<Steps>
  <Step title="Build the Image">
    ```bash
    docker build -t mirofish:local .
    ```
  </Step>

  <Step title="Update docker-compose.yml">
    Edit `docker-compose.yml` to use your local image:

    ```yaml
    services:
      mirofish:
        image: mirofish:local  # Use local image
        # image: ghcr.io/666ghj/mirofish:latest
    ```
  </Step>

  <Step title="Start the Container">
    ```bash
    docker compose up -d
    ```
  </Step>
</Steps>

## Troubleshooting

### Container Fails to Start

**Check logs for errors:**

```bash
docker compose logs
```

**Common issues:**

- **Missing environment variables**: Ensure `.env` file exists and contains required keys
- **Port conflicts**: Check if ports 3000 or 5001 are already in use
  ```bash
  # On Linux/macOS
  lsof -i :3000
  lsof -i :5001
  
  # On Windows
  netstat -ano | findstr :3000
  netstat -ano | findstr :5001
  ```

### Frontend Not Loading

**Verify the container is running:**

```bash
docker compose ps
```

**Check frontend logs:**

```bash
docker compose logs | grep frontend
```

**Try restarting the container:**

```bash
docker compose restart
```

### Backend API Errors

**Verify API keys are correct:**

1. Check your `.env` file for correct API keys
2. Verify the LLM API key at [Alibaba Bailian Console](https://bailian.console.aliyun.com/)
3. Verify the Zep API key at [Zep Cloud](https://app.getzep.com/)

**Restart after updating environment variables:**

```bash
docker compose down
docker compose up -d
```

### Slow Image Download

If pulling the image is slow:

1. Edit `docker-compose.yml`
2. Uncomment the mirror registry line:
   ```yaml
   image: ghcr.nju.edu.cn/666ghj/mirofish:latest
   ```
3. Run `docker compose pull` again

### Permission Errors with Uploads

If you see permission errors when uploading files:

```bash
# Fix permissions on the uploads directory
chmod -R 777 backend/uploads
```

### Container Uses Too Much Memory

Limit container resources by editing `docker-compose.yml`:

```yaml
services:
  mirofish:
    image: ghcr.io/666ghj/mirofish:latest
    deploy:
      resources:
        limits:
          memory: 4G
          cpus: '2'
```

### Reset Everything

To completely reset the installation:

```bash
# Stop and remove containers, networks, and volumes
docker compose down -v

# Remove the image
docker rmi ghcr.io/666ghj/mirofish:latest

# Start fresh
docker compose up -d
```

<Warning>
  Using `-v` flag will delete all volumes, including uploaded files in `backend/uploads/`. Back up important data first.
</Warning>
