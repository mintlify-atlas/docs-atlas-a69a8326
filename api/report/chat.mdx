## Overview

The Report Chat API enables interactive conversations with the Report Agent. The agent can autonomously retrieve information from the knowledge graph to answer questions about the simulation results and generated reports.

<Info>
The chat agent uses the same ReACT (Reasoning + Acting) architecture as report generation, but optimized for conversational responses.
</Info>

## Chat with Report Agent

### POST /api/report/chat

Send a message to the Report Agent and receive an intelligent response with optional tool usage.

**Request Body**

<ParamField body="simulation_id" type="string" required>
  The simulation ID to query about
</ParamField>

<ParamField body="message" type="string" required>
  User's question or message
</ParamField>

<ParamField body="chat_history" type="array">
  Conversation history for context (optional)
  
  Each item should be an object with:
  - `role`: "user" or "assistant"
  - `content`: Message content
</ParamField>

**Response**

<ResponseField name="success" type="boolean">
  Whether the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  <ResponseField name="response" type="string">
    Agent's response to the user's message
  </ResponseField>
  
  <ResponseField name="tool_calls" type="array">
    List of tools the agent called to answer the question
    
    <ResponseField name="name" type="string">
      Tool name (e.g., `insight_forge`, `panorama_search`, `quick_search`, `interview_agents`)
    </ResponseField>
    
    <ResponseField name="parameters" type="object">
      Parameters passed to the tool
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="sources" type="array">
    Information sources used (query strings from tool calls)
  </ResponseField>
</ResponseField>

```python Python
import requests

class ReportChatClient:
    def __init__(self, simulation_id: str):
        self.simulation_id = simulation_id
        self.history = []
    
    def ask(self, message: str) -> str:
        response = requests.post(
            "https://api.mirofish.com/api/report/chat",
            json={
                "simulation_id": self.simulation_id,
                "message": message,
                "chat_history": self.history
            }
        )
        
        data = response.json()["data"]
        
        # Update history
        self.history.append({"role": "user", "content": message})
        self.history.append({"role": "assistant", "content": data["response"]})
        
        # Show tool usage
        if data["tool_calls"]:
            print(f"[Agent used {len(data['tool_calls'])} tool(s)]")
        
        return data["response"]

# Usage
chat = ReportChatClient("sim_abc123")
response = chat.ask("学生群体的主要反应是什么？")
print(response)
```

## How the Agent Works

### Agent Capabilities

The Report Agent operates with the following workflow:

1. **Context Loading**
   - Loads existing report content (if available)
   - Reads simulation requirements
   - Maintains conversation history

2. **Intent Analysis**
   - Determines if question can be answered from report
   - Identifies if additional data retrieval is needed

3. **Tool Selection** (if needed)
   - Autonomously chooses appropriate tools
   - Formulates queries to retrieve relevant information
   - Limited to 1-2 tool calls per response for efficiency

4. **Response Generation**
   - Synthesizes information from report and tool results
   - Includes direct quotes from simulation data
   - Provides concise, clear answers

### Available Tools

The agent can use these tools to answer questions:

**insight_forge**
- Deep analysis of specific topics
- Multi-dimensional knowledge graph search
- Decomposes complex questions into sub-queries

**panorama_search**
- Get complete event timeline
- Understand evolution of public opinion
- Distinguish current vs. historical facts

**quick_search**
- Fast fact lookup
- Simple information retrieval
- Verify specific details

**interview_agents**
- Get first-person perspectives from simulated agents
- Interview specific roles (students, media, officials, etc.)
- Dual-platform interviews (Twitter + Reddit)

<Note>
The agent prioritizes answering from the existing report first, only calling tools when the report doesn't contain sufficient information.
</Note>

### Response Optimization

The chat agent is optimized for conversational efficiency:

- **Fast responses**: Answers from report content without tool calls
- **Limited tool usage**: Maximum 1-2 tool calls per message
- **Concise answers**: Direct and to-the-point responses
- **Contextual memory**: Maintains conversation history (last 10 messages)

## Chat Context Management

### Conversation History

The agent maintains conversation context through the `chat_history` parameter:

```python
# Initialize empty history
chat_history = []

# First message
response1 = chat("这次模拟的背景是什么？", chat_history)
chat_history.append({"role": "user", "content": "这次模拟的背景是什么？"})
chat_history.append({"role": "assistant", "content": response1["response"]})

# Follow-up message with context
response2 = chat("学生的主要诉求有哪些？", chat_history)
chat_history.append({"role": "user", "content": "学生的主要诉求有哪些？"})
chat_history.append({"role": "assistant", "content": response2["response"]})
```

### History Limits

- Maximum 10 previous messages retained
- Older messages automatically truncated
- Report content limited to 15,000 characters

### Best Practices

1. **Maintain history** for multi-turn conversations
2. **Clear history** when starting a new topic
3. **Ask specific questions** for better responses
4. **Reference previous context** in follow-up questions

## Tool Usage Patterns

### When Tools Are Called

The agent decides to use tools when:

- Question requires detailed information not in the report
- User asks for specific data or statistics
- Question involves agent perspectives or quotes
- Timeline or evolution analysis is needed

### When Tools Are Not Called

The agent answers directly from report when:

- Question can be answered from existing report content
- User asks for summary or overview
- Question is about report structure or metadata
- Simple clarification is needed

### Tool Selection Logic

```typescript
// Agent's decision process
if (questionInReport) {
  return answerFromReport();
} else if (needsDeepAnalysis) {
  useInsightForge();
} else if (needsTimeline) {
  usePanoramaSearch();
} else if (needsAgentPerspective) {
  useInterviewAgents();
} else {
  useQuickSearch();
}
```

## Performance Considerations

### Response Times

- **No tools**: 1-2 seconds (direct from report)
- **1 tool call**: 3-5 seconds (single retrieval)
- **2 tool calls**: 5-8 seconds (multiple retrievals)

### Optimization Tips

1. **Generate report first**: Chat works best with completed reports
2. **Ask focused questions**: Specific questions get faster responses
3. **Use conversation history**: Helps agent understand context
4. **Avoid overly broad questions**: May trigger multiple tool calls

<Warning>
The chat agent requires an existing simulation with knowledge graph. Report generation is recommended but not required - the agent can answer questions using direct knowledge graph queries if no report exists.
</Warning>
